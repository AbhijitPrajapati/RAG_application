# rag_application
This simple command-line tool enables RAG (retrieval augmented generation) leveraging local CUDA acceleration for private, customizable inference. It accepts plain text documents and allows flexibility in model choice to meet hardware constraints. Additionally, it employs numerous methods including semantic chunking and reranking in order to improve the effectiveness of the local RAG pipeline. The tool allows multi-turn conversation as well, taking advantage of enhanced reasoning capabilities. 

## Requirements
### Hardware
- NVIDIA GPU (Maxwell archetecture or newer, compute capability >= 5.0)
- see https://developer.nvidia.com/cuda/gpus and https://developer.nvidia.com/cuda/gpus/legacy for details on compute capability
### Software
- Python 3.10+
- CUDA 13.0
- Compatible NVIDIA Drivers

## Installation
### 1. Clone the Repository
~~~
git clone https://github.com/AbhijitPrajapati/rag_application
cd rag_application
~~~
### 2. Install Dependencies
~~~
pip install -r requirements.txt
~~~
## Model Setup
1. Download a GGUF model 
2. Place it in `/gguf` or another location

## Configurations
### 1. Configure `gpu_model_config.json`
- n_gpu_layers: number of layers to be loaded onto the GPU (maximum capacity depends on hardware)  
- n_threads: number of CPU threads used (match roughly with number of cores)  
- n_batch: number of tokens processed per batch  
- model_path: path to the GGUF file  
- chat_format: chat format associated with the selected model
  - accepts any chat format allowed by llama-cpp-python

### 2. Configure `rag_config.json`
- n_chunks: number of chunks initially retrieved
- top_n: number of chunks utilized after reranking
- temperature: temperature of the large language model (variability)
- max_tokens: maximum number of tokens generated by the model (null = no maximum)
- top_p: top_p sampling parameter (lower = more consistency)
- top_k: top_k sampling parameter (higher = more variablity)

## Usage
### Prompting the Model
~~~
python main.py chat
~~~
### Adding Documents
~~~
python main.py documents ingest <file or folder paths>
~~~
### Viewing or Deleting Documents
~~~
python main.py documents manage
~~~
Navigate the table with arrow keys and use enter to select a document for deletion. Use "d" to begin deletion and "q" to exit.
